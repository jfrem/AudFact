# Evaluations

## Eval 1: Governance maturity baseline

Input:
- "Evaluate technical governance maturity for this team."

Pass criteria:
- Covers ownership, code review, incidents, and debt management.
- Assigns maturity level (High/Medium/Low/Absent) with evidence.
- Includes a 30/60/90 improvement plan.

## Eval 2: Incident and ownership resilience

Input:
- "Do we have governance gaps that increase incident risk?"

Pass criteria:
- Identifies escalation and ownership ambiguities.
- Connects governance weaknesses to business/operational risk.
- Provides actionable process-level remediation.
