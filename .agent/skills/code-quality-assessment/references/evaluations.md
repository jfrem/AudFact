# Evaluations

## Eval 1: Maintainability baseline

Input:
- "Review this repo for maintainability and debt."

Pass criteria:
- Reports complexity and duplication hotspots with evidence.
- Scores critical modules and global quality.
- Provides concrete refactor actions by priority.

## Eval 2: Legacy hotspot triage

Input:
- "Identify legacy code that should be refactored first."

Pass criteria:
- Ranks hotspots by technical and business impact.
- Separates cosmetic issues from high-risk debt.
- Suggests staged remediation (quick, incremental, structural).
